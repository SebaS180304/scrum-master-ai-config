# How to contribute

## Steps

### 1. Issue

Create an Issue in Github Projects with all the correct tags and assign it to yourself.

### 2. Branch

Create a new branch following the naming format: `<area>/<description>`.

### 3. Pull Request

Create a Pull Request (PR) with the base branch as main. You may publish the PR as draft while you work on the feature, but only open it for review once you believe it's complete, tested and ready to merge. Be sure to provide a small description providing context for reviewers and an explanation of the tests you ran.

### 4. Reviewing

Everyone on the development team may review a PR. Be sure to provide suggestions to improve the quality of the code, ask for changes if the feature isn't what was intended, and ask questions if something isn't clear. For an approve stamp being granted the resulting code must be functional, readable and mantainable.

### 5. Merging

The reviewers shouldn't merge a PR unless it's time sensitive. Authors should always merge their own PRs as a good practice.

## Format

Use the Prettier VS Code extension to format every file on you PRs. This will improve readability and consistency accross the code base.
